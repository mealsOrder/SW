문제
문제명 : 순서쌍
제한 조건

실행시간 : 50개의 테스트 케이스를 합쳐서 C/C++의 경우 1초, JAVA의 경우 1초, Python 2초

메 모 리 : Heap, Global, Stack 등을 모두 합해 최대 256MB까지 사용 가능 (단, 스택은 최대 1MB까지 사용 가능)

제출 횟수 제한

제한 없음

채점

Accept : 정상적으로 제출되었다는 의미 (컴파일 에러 등 없다는 의미이며 점수는 공개하지 않음)

Fail : 컴파일, 런타임 에러 등으로 0점

평가

sample_input은 편의를 위해 제공하며, 실제 채점은 별도의 평가용 input으로 이루어진다.


 

N개의 자연수가 있는 수열이 주어진다.

정욱이는 수열의 모든 가능한 쌍을 나열하고 정렬하였을 때 K번째에 오는 쌍을 구해서 그 원소들의 합을 출력하고 싶다

여기서 모든 가능한 쌍이라 함은 순서쌍의 첫 번째 값이 수열의 모든 값이 될 수 있고,

두 번째 값도 수열의 모든 값이 될 수 있으며, 중복을 모두 포함한 N^2개의 순서쌍을 말한다.

순서쌍 (a, b)들을 정렬한다는 것은 a가 작은 순서쌍들이 먼저 배열되고,

a가 같은 경우 b가 작은 순서쌍들이 먼저 배열되는 것을 말한다

 

아래와 같이 입력이 주어졌다

수열은 3 2 3 이며 N은 3이고, K는 7이다


이 예시에서 모든 순서쌍을 정렬하여 나열하면 다음과 같다.



1번째 (2, 2)

2번째 (2, 3)

3번째 (2, 3)

4번째 (3, 2)

5번째 (3, 2)

6번째 (3, 3)

7번째 (3, 3)

8번째 (3, 3)

9번째 (3, 3)

따라서 7번째 순서쌍은 (3, 3)이고 원소들의 합은 6이다.



다른 경우를 보자

주어진 수열이 1, 2, 3, 4, 5 이고, K는 6 이다

1번째 (1, 1)

2번째 (1, 2)

3번째 (1, 3)

4번째 (1, 4)

5번째 (1, 5)

6번째 (2, 1) 이 되고, 이 때 원소의 합은 3 이다

...

  

[제약사항]

1.     수열의 길이 N은 3 이상 100 이하이다.

2.     수열의 원소 값은 1 이상 200 이하이다.

3.     K는 1 이상 N의 제곱 이하이다.




[입력]

가장 첫 줄에는 테스트 케이스의 총 수가 주어진다.

그 다음 줄부터 각 테스트 케이스가 주어지며, 각 테스트 케이스는 2줄로 구성된다.

각 테스트 케이스의 첫 줄에는 N이 주어진다. 다음 줄에는 수열이 주어진다.

 

[출력]

출력의 각 줄은 ‘#x’로 시작하고, 공백을 한 칸 둔 후, K번째에 오는 쌍의 원소 합을 출력한다. 단, x는 테스트 케이스의 번호이다.



[입력 예]

3                         // 테스트 케이스의 수


3 7                       // N = 3, K = 7테스트 케이스 #1


3 2 3


5 6                       // N = 5, K = 6 테스트 케이스 #2


1 2 3 4 5


5 10                       // N = 5, K = 10 테스트 케이스 #3


3 2 1 1 3

 

[출력 예]

#1 6


#2 3


#3 4
