접근방법 구상

1. 완전히 새로운 문제는 없다
-유사, 특정 자료구조, 전형적 알고리즘

2. 문제의 설명대로 시도 
-TC 입력 -> 출력을 손으로 직접해본다 >> 규칙성 발견

3. 규칙성을 발견하기 힘들다면
- 가능한 모든 경우를 처리 (시간,메모리 문제가 있을 수 있음)
- 일부분으로 쪼개기
- 반대로 생각해보기

4. 모든 구상을 시각적으로(손)


문제 풀이 연습 팁

1.기본기는 철저히 연습(틀, 루프, 범위)
	1) 기본기가 탄탄해야 구상한 아이디어 실수 없이 구현
	2) 정확한 입출력, 실수 없는 2차원 배열 처리 및 다중 루프 구현
	3) 가장 효율적인/짧은/멋있는 코드보다 기본적인 [반본/조건]을 빈틈없이 구현!
	4) 손코딩 : 사용하는 주요 array, 범위, 핵심코드를 실명/시각적으로 설계하고 접근
	5) 필요한 위치에서 필요한 디버깅 가능하도록 디버거 사용법 숙지
	6) 문제읽기 -> 접근방법 구상 -> 핵심코드 손코딩(키보드를 멀리하세요)

2. 나만의 환경/루틴으로 심리적 안정 (맨탈관리)
	1) 파이참 환경, 폰트, 폰트크기, 창 배치, TC입력파일, A4용지, 풀이 순서
	2) 익숙한 이름: 입력 받는 변수, 선언한 변수, 특정용도 사용 변수 등
	3) 함수호출, 조건, 반복, break, continue 등 익숙한 방법으로 구현
